# 击退系统设计（计划）

> 状态：**计划中，待讨论后实现**。本文记录方案与待定问题，尚未动代码。
> 关联：[[建筑设计.md]] A 树 A31Bomb（加农炮）。

---

## 1. 目标

为塔的攻击引入"击退/位移"效果：爆炸/冲击波把敌人**沿"爆心质心 → 敌人质心"的射线方向**推开，位移速度随时间**衰减**到 0，然后敌人平滑回到原路线继续走。

**首个用例**：A31Bomb 加农炮——落点爆炸，范围伤害 + 冲击波击退。
此系统设计为**通用基础设施**，后续黑洞（反向吸引）、核弹等也复用。

---

## 2. 核心结论

- **寻路算法（Theta\*）完全不受影响**：它只在地图变化（放/拆障碍）时运行，击退不触发重新寻路。
- 相关的是**路径跟随层 `PathFollower`**，它每帧用"真实位置重新投影到路线"来决定走向（不是死记路点索引），所以**被外力推开后能自动、平滑地走回路线**——recovery 基本免费。
- 击退是"衰减速度"型，与 `PathFollower` 现在的 `MoveTowards`（无动量、纯位置）正交，叠加一个独立衰减速度即可，二者不冲突。

→ **架构支持，且无需改寻路。** 只需在跟随层补三块。

---

## 3. 实现方案（三块，集中在 `PathFollower.cs`）

### 3.1 击退速度通道（新增速度概念）
当前 `PathFollower` 无速度状态，直接由路线设置位置。新增：

```
private Vector3 knockVel;

public void AddKnockback(Vector3 sourcePos, float strength)
{
    Vector3 dir = (transform.position - sourcePos);
    dir.z = 0;
    knockVel += dir.normalized * strength;   // 叠加，支持多次命中
}

// Update 末尾，在路径跟随移动之后：
transform.position += knockVel * Time.deltaTime;
knockVel = Vector3.MoveTowards(knockVel, Vector3.zero, knockDecay * Time.deltaTime);
// （或指数衰减 knockVel *= Mathf.Exp(-k*dt)，手感待定）
```
- `knockDecay`（或衰减系数 k）做成可序列化，调手感。
- 推开 → 衰减 → 路径跟随把它拉回，一气呵成。

### 3.2 墙体钳制（**最关键，否则会穿墙**）
墙不是物理碰撞体，敌人是 trigger + kinematic，**没有任何东西阻止被推进墙里**。施加位移时必须钳制：
- 新位置若落入不可走格 → 停在可走边界 / 沿墙滑动（分轴尝试：先试 x 位移、再试 y 位移，能走的轴才动）。
- 保证圆心始终在可走格内（理想：留敌人半径余量，避免身体嵌墙）。

### 3.3 双向重投影（击退期间）
`AdvanceSegment` 现在是"只向前找（窗口 4 段）"。冲击波可能把敌人**往入口方向推**：
- 只向前找会导致进度（`seg` / `DistanceToExit`）失真、回退时擦墙。
- 方案：击退激活期间（`knockVel` 非零）改为**双向 / 全局重投影**重新锁定最近段；击退结束后恢复正常向前窗口。

## 5. 待讨论 / 开放问题

- **衰减曲线**：线性衰减 vs 指数衰减，哪个手感好？击退总距离/时长大概多少？
- **强度模型**：固定强度，还是随距爆心距离衰减（越近推得越远）？
- **墙体处理**：撞墙是"急停"还是"沿墙滑行"？滑行更顺但实现略复杂。
- **击退中是否可被再次寻路打断**：击退期间若玩家放障碍触发 `Recompute`，如何与 `knockVel` 协调（建议：照常重算路线，击退速度独立叠加，不冲突）。
- **眩晕/减速联动**：A31Injection 等有减速，击退期间是否暂停前进只保留位移？
- **视觉**：是否要击退时的拖拽动画 / 冲击波特效（本期可不做）。
